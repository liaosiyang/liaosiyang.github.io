Android笔记-动画机制

2016-09-13
  • Android笔记
  • Android系统

以前在学习Cocos2d-x对它丰富的动画机制很是欣赏，游戏引擎中动画系统是重头戏。在平时的Android开发中有时也需要用到动画效果，比如之前做的点菜功能会有一个菜品飘到购物车里的一个动画。
再比如现在手机上直播给主播点赞的话会有小星星飘出来，这也用到了动画。动画效果让App更富有活力和表现力。
总结一下Android动画相关知识。

补间动画(Tween Animation)

补间动画分为四类：

  1. TranslateAnimation平移动画，效果是移动View;
  2. ScaleAnimation缩放动画，效果是放大或缩小View;
  3. RotateAnimation旋转动画，效果是旋转View;
  4. AlphaAnimation透明度动画，效果是改变View的透明度；

补间动画一般在res/anim中声明，可以声明单个动画，也可以声明一系列动画。
< set >标签表示动画集合，可以包含若干个动画，内部也是可以嵌套其他动画集合。
< set >中可以指定插值器来控制动画播放的速度，比如加减速播放插值器，还可以指定集合中的动画是否共享一个插值器。

除了可以在XML文件中定义动画外，还可以通过Java代码来定义。

帧动画(Frame Animation)

帧动画是按照顺序播放一组图片。通常在XML文件中声明一个< animation-list >标签按顺序指定每一帧的图片，然后作为View的背景就行了。
可以设定设置播放次数，也可以设定循环播放。
帧动画使用比较简单，但容易引起OOM，所以在使用帧动画的时候要避免使用尺寸较大的图片。

属性动画(Property Animation)

属性动画是Android3.0加入的，属性动画的效果是将一个对象的属性值在一定时间内转变成另一个属性值，注意是对象而不是View。
比如对象的背景色属性，View的backgroundColor，就可以在指定时间内由红色变成蓝色。同样还有rotation、translation、scale、alpha等，就可以实现旋转、平移、缩放、透明度等属性的变化。
属性动画要求对象的属性必须拥有get、set方法，实际上属性动画就是根据传递的该属性的初始值和最终值，和设定的动画效果不断地调用set方法。

可以使用Java代码实现，也可以通过XML文件定义，和补间动画一样，也是放在< set >标签中。实际上补间动画完全可以由属性动画取代，只需注意兼容Android3.0以下版本的问题。

插值器和估值器

插值器作用是控制动画的播放速度。
系统预置的插值器有：

  1. LinearInterpolator线性插值器：匀速动画
  2. AcclerateDecelerateInterpolator加速减速插值器：动画两头慢中间快
  3. DecelerateInterpolator减速插值器：动画越来越慢

估值器作用是计算动画播放时的属性值。
系统预置的估值器有：

  1. IntEvaluator 整型估值器
  2. FloatEvaluator 浮点型估值器
  3. ArgbEvaluator ARGBcolor值估值器

插值器和估值器联合使用是实现非匀速动画的手段，比如实现一个小星星从屏幕底部飘到顶部并且飘动的速度越来越快(插值器)，路径是贝塞尔曲线(估值器)。

监听器

属性动画还可以使用监听器监听动画的播放过程，主要有2个接口：AnimatorUpdateListener和AnimatorListener。
AnimatorUpdateListener可以监听动画的开始、结束、取消以及重置播放。
AnimatorListener可以到监听动画每一帧。

使用动画注意事项

  1. OOM：帧动画容易出现OOM问题，尽量避免使用较多且较大的图片。
  2. 内存泄漏：属性动画有无限循环的类型，应该在Activity退出时停止，否则将导致Activity无法释放。补间动画不存在此问题。
  3. 硬件加速：为了提高动画的流畅度，建议开启硬件加速。
